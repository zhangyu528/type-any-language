## 1. 产品概述

`type-any-language` 是一个**单页 Web 应用**，围绕一个核心循环：

> **选词库 → 听句子音频 → 逐字输入完整句子 → 即时反馈 → 继续下一题**

整个体验只服务**「听音写句」（dictation）**这一种练习模式：

- 无注册、无登录、无后端写入
- 无关卡、无积分商城、无社交
- 用户可以选「哪本词库 + 哪个难度」练，但练习本身只有这一种形式
- 所有内容（词库、句子、音频）在后端烤入；前端是只读 + 客户端校验的薄壳

**目标用户**：已经掌握基础英语、想用「听 → 写」的方式巩固真实句子的学习者。

## 2. 用户与目标

### 2.1 目标用户

- 已经掌握基础英语（具备听写基本句子的能力）
- 想要**在真实句子语境中**巩固词汇和语法
- 时间碎片化（通勤、午休、睡前），希望快速进入 / 退出练习
- 不喜欢被打扰、不喜欢被催促

### 2.2 用户的核心需求

| 需求 | 产品应对 |
|---|---|
| 选自己想练的词库和难度 | 顶部词库选择器 |
| 听清句子 | 播放控制条（倍速 / 循环 / 重播） |
| 知道自己哪里对、哪里错 | 字符级即时反馈 + 逐格自动校验 |
| 卡住时能看答案 / 提示 | 答案提示、显示完整句子 |
| 了解自己做得怎么样 | 题内进度圆点 + 终局得分页 |
| 反复练、调整节奏 | 倍速、循环、输入模式、关闭提示音 |

### 2.3 用户不需要的

- 注册、登录、社交、个人主页
- 积分、徽章、排行榜、成就系统
- 课程包、付费内容、订阅
- 强引导、教学引导、首次使用教程
- 推送通知、邮件提醒

> **产品基调**：安静地、专注地、反复地做这一件事。不打扰用户。

## 3. 核心使用流程

### 3.1 首次访问

```
打开页面
  ↓
后端拉取「词库目录」 → 看到默认词库 + 难度
  ↓
后端拉取该 (词库, 难度) 的 10 句练习
  ↓
进入练习页：第 1 题
```

### 3.2 单题循环

```
1. 看到中文提示（题目的中文意思）
2. 点波形 / 播放按钮 → 听音频
3. 听不清可点循环 / 调倍速 / 重播
4. 逐字输入听到的英文
   - 每输入一个字符：用户能看到这个字符「对 / 错」
   - 输完一个词：自动跳到下一格
   - 最后格输对：短暂延迟后自动提交
5. 系统判定：
   - 答对：播答对音 + 立即切下一题（不显示反馈）
   - 答错：显示反馈卡，含正确答案 + Next 按钮
6. 10 题做完（或中途按 Esc） → 得分页
```

### 3.3 中途离开再回来

| 维度 | 是否被记住 |
|---|---|
| 上次选的词库 / 难度 | ✅ 是 |
| 倍速 | ✅ 是 |
| 输入模式（按序 / 自由） | ✅ 是 |
| 答对提示音开关 | ✅ 是 |
| 快捷键面板是否展开 | ✅ 是 |
| 工具菜单（设置 / 主题 / 引导） | N/A（无内容） |
| **练习进度本身** | ❌ **否 —— 刷新就重置** |

> **产品决策**：进度不持久化是有意为之 —— 鼓励「一次性专注完一段练习」，不是「随时继续」。这降低了用户的认知负担（不需要想"我刚才做到哪了"）。

## 4. 功能需求

> 每个功能按「**用户目标 → 行为规则 → 信息显示 → 设计含义**」四段写。
> - **行为规则** 是硬性约束（必须遵守）
> - **设计含义** 是给设计师的提示（指导设计方向，但**不限定具体视觉**）

### 4.1 词库选择

**用户目标**：选想练的词库 + 难度桶。

**行为规则**：
- 必须能从可见选项中选择一个词库 + 一个难度
- 切换词库时，若当前难度在该词库下也可用则保留；否则自动切到该词库的默认难度
- 加载中时控件不可操作
- 失败时显示错误信息，但**不阻塞**页面其他区域

**信息显示**：
- 每个词库的名称 + 词数
- 当前选中的状态必须能被用户一眼识别
- 加载中、加载失败两种状态

**持久化**：选过的词库和难度被记住。如果上次的词库被运营下线，自动回退到目录的第一项。

**设计含义**：
- 用户需要快速识别"我在练什么"
- 选中态需要明显但不喧宾夺主（视觉对比 vs 视觉重量的平衡）
- 切换词库应该是低成本的探索行为 —— 不要让用户觉得"切错了就回不来了"

---

### 4.2 音频播放

**用户目标**：听清句子，可控制播放速度。

**行为规则**：
- 必须能从静止状态主动触发播放
- 必须能暂停
- 倍速必须在 **3 档** 之内：0.5x / 1x / 2x
- 倍速切换**不打断播放**（不重置音频位置、不重新加载）
- 必须能切换循环播放开关
- 浏览器自动播放策略必须被尊重 —— 第一次播放需要用户主动触发
- 切到新题时播放状态重置

**信息显示**：
- 当前播放 / 暂停状态
- 当前倍速
- 当前是否循环
- 当前题号 / 总题数

**持久化**：倍速被记住。

**设计含义**：
- 播放控制是**核心操作**之一 —— 视觉权重应高于次要控件
- 播放状态变化应有即时视觉反馈（不能"点了没反应"）
- 循环 / 倍速等次要控件可以更克制

---

### 4.3 句子输入

**用户目标**：把听到的英文**逐字**敲出来，过程中就能看出对错。

**行为规则**：
- 整句被按空格切分成多个**格子**（cell），每个英文单词一格
- 屏幕上有且仅有一个聚焦的输入机制（用户的所有按键都进这里）
- 字符级即时反馈：用户输入的每个字符 vs 标准字符的对错
- 输完一格 + 全部正确 → 自动跳到下一格
- 最后一格输对 → 短暂延迟（300ms）后自动提交
- 空格键作为「跳到下一格」和「提交」的统一触发
- 答错时（空格键触发完整 + 仍有错）→ 给出**视觉提示**（如震动 / 闪烁 / 标记），不切到下一格

**容错规则**（系统对答案的宽松匹配，**硬性**）：
- 大小写不敏感
- 标点符号（`.` `,` `!` `?` `;` `:` `'` `"`）忽略
- 连续空白折叠成单空格
- 头尾空白忽略

**信息显示**（任何时刻必须可见）：
- 整句的所有格子
- 当前光标所在的格
- 已答对的格（视觉上与未答的格**不同**）
- 当前格的字符级对错状态

**设计含义**：
- 字符级反馈的"颗粒度"很细 —— 设计师要选一种"不刺眼但可读"的对错呈现方式（颜色 / 形态 / 装饰元素等均可）
- 答错的字符需要**保留显示**（不要被覆盖），让用户看到错在哪个位置
- 输入的"光标"位置必须明确，但不应该遮挡文本
- 切换格子时**布局不能跳动**（用户已经在视觉上"瞄准"了某个位置）

---

### 4.4 两种输入模式

**用户目标**：选择符合自己节奏的输入方式。

| 模式 | 行为 | 适合 |
|---|---|---|
| **按序输入**（默认） | 答对自动跳下一格 | 想「先听后写、连续敲下去」的沉浸节奏 |
| **自由点选** | 答对不跳；用户可主动切换任意格（Tab / Shift+Tab / 点击） | 想「先标记不会的、逐个攻克」的解题节奏 |

**行为规则**：
- 模式可被用户切换
- 切换是即时的（不需要确认）
- 在中文输入法拼写中时，点击切格不应误触发

**信息显示**：当前模式必须对用户可见（位于句子输入区附近）

**持久化**：所选模式被记住。

**设计含义**：
- 自由模式下，每个未答对的格都应能被点击 —— 视觉上要传达"这里可以点"
- 模式选择器本身应该**不抢戏**（不是练习的主角）

---

### 4.5 答案提示（Hint）

**用户目标**：卡住时看一眼当前格的标准答案。

**行为规则**：
- 可由用户主动触发（具体按键见 §6 交互规则）
- 提示只显示**当前格**的答案，不显示整句
- 提示出现后，用户开始输入或再次触发就消失
- 触发方式必须在「正在输入的格内」和「不在输入格内」两种场景下都可用

**信息显示**：
- 当前格的正确单词
- 提示出现时有淡入动画
- 消失时有淡出动画

**设计含义**：
- 提示是**辅助**功能，不是奖励 —— 视觉上应比正文更轻
- 不应该让用户"为了看提示而分心" —— 触发方式要简单

---

### 4.6 显示完整句子（救命功能）

**用户目标**：彻底卡死时一次性看答案，然后从中断处继续。

**行为规则**：
- 可由用户主动触发
- 打开时：整句直接铺在格子上（所有格显示标准答案）
- 打开时：当前输入状态被「快照」保存
- 关闭时：自动恢复到**最后一个连续答对的格**之后的位置（不是从头来）
- 关闭时：未答对的格子输入被清空，但**已答对的格保留**

**信息显示**：
- 开关状态（开 / 关）
- 开启时，整句的所有字符都按「正确」样式展示

**持久化**：否（每次进入新题时重置为关）

**设计含义**：
- 开关位置应在「设置类」控件附近（不是练习主流程上）
- 视觉上不鼓励频繁使用 —— 但当用户需要时**一秒就能找到**

---

### 4.7 答对提示音

**用户目标**：在答对时获得一个**简短的正反馈**。

**行为规则**：
- 答对瞬间播一个**音频反馈**
- 必须可关闭
- 默认开启
- 偏好被记住

**设计含义**（重要 —— 这是**产品调性**的体现）：
- 声音的"性格"应该符合产品基调：克制 / 不打扰 / 不夸张 / 不胜利式
- ❌ 不应是"胜利号角""撒花音效""🎉 恭喜"类
- ✅ 应是"三音短旋律"或"一声轻响"或类似的内敛反馈
- 音量克制（不刺、不抢注意力）
- 时长短（< 1s）

> 设计师可以重新设计这段声音，但**性格要求**不变：安静、不喧宾夺主。

---

### 4.8 显示音标

**用户目标**：辅助理解单词读音。

**行为规则**：
- 可由用户主动开启
- 开启时：每个单词格下方显示对应的 IPA 国际音标
- 音标加载在后台异步完成
- 同一个单词在一次会话内**只需加载一次**（跨题目复用）
- 加载未完成的格**不阻塞**练习

**信息显示**：
- 每个格下方的音标（如 `/əˈpærəntli/`）
- 加载中的占位标记
- 加载失败的占位标记

**持久化**：否（每次进入新题时按默认值重置）

**设计含义**：
- 音标是**辅助信息**，视觉权重应低于单词本身
- 占位标记应低调（避免"加载中"图标喧宾夺主）

---

### 4.9 进度与计分

#### 4.9.1 题内进度

**用户目标**：知道"现在做到第几题、整体进度"。

**行为规则**：
- 一题一个进度标记
- 三个状态必须可区分：当前题 / 已答对 / 已答错
- 10 题的进度应在用户视觉可达范围内（不需要滚动）

**设计含义**：
- 进度标记的视觉重量**不能比句子本身重**（进度是次要的，句子是主角）
- 当前题应"亮"但不"喊"

#### 4.9.2 终局得分

**用户目标**：完成一轮后看看自己做得怎么样。

**行为规则**：
- 10 题完成 → 自动跳到得分页
- 中途按 `Esc` → 也跳到得分页（按当前已答情况计）
- 得分页必须包含：分数（如 `7 / 10`）+ 鼓励文案 + "再来一次"的入口
- 鼓励文案应**有分档**（全对 / 全错 / 部分正确 文案不同），但**不评级、不比较、不批评**

**设计含义**：
- 得分页是**仪式感**的体现，不是成绩单
- 视觉重心应该是"鼓励"而不是"分数"
- "begin again"按钮必须有，但不应是"再来挑战！"式的视觉强调

#### 4.9.3 文案调性分档

| 分数 | 文案示例 |
|---|---|
| 全对 | `a complete breath.` |
| 全错 | `the beginning of attention.` |
| 部分正确 | `7 of 10 — quiet progress.` |

> 调性是产品决策（**不评分、不比较、不催促**），但具体用词可以由设计师替换。

---

### 4.10 快捷键面板

**用户目标**：一眼看到所有键盘操作；切换显示选项。

**行为规则**：
- 可由用户主动展开 / 折叠
- 面板内必须包含**快捷键列表** + **显示选项复选框**
- 复选框必须可勾选：显示句子、显示音标、答对提示音
- 点击面板外或按 Esc → 关闭
- 移动端（屏幕宽度极窄时）可隐藏整个面板

**信息显示**：
- 快捷键：键名 + 行为描述
- 显示选项：当前开 / 关状态

**持久化**：面板是否展开被记住。

**设计含义**：
- 面板是**信息密度较高**的区域 —— 但因为只在用户主动打开时显示，可以承担更密集的内容
- 不应该"挡住"主练习区（浮层 / 抽屉式，避免页面跳动）

---

### 4.11 工具菜单

**位置**：右上角的 `…` 按钮。

**当前菜单项**：

| 菜单项 | 状态 |
|---|---|
| 设置 | 占位（产品上规划中） |
| 主题 | 占位（产品上规划中） |
| 功能引导 | 占位（产品上规划中） |

**产品决策**：
- 三个菜单项**当前只触发一个「功能待实现」的提示框**
- 这意味着用户在产品中能看到入口，但点开后发现没有内容
- 这是**有意的** —— 让用户感知到"产品在持续生长"比"隐藏掉一个不存在的功能"更好
- 它们不在主流程的视线焦点上，**不会造成干扰**

**设计含义**：
- `…` 按钮应**克制**（不抢练习主流程的视觉权重）
- 弹出的「功能待实现」提示应是轻量的（不打断用户）

---

### 4.12 导航

**用户目标**：在题目间穿梭、随时查看得分。

**行为规则**：
- 必须能从当前题切到上一题 / 下一题
- 第一题时「上一题」不可用
- 最后一题时「下一题」跳到得分页
- 用户主动操作（点击 / 按键）后，导航提示短暂显示后自动隐藏
- `Esc` 始终可立即跳到得分页

**信息显示**：
- 当前题号 / 总题数
- 上一题 / 下一题的可点状态
- 提示用户「按 Esc 看得分」的文字（用户操作后短暂显示）

**设计含义**：
- 导航提示是**辅助** UI，应该"轻"且"短命"
- 题目切换时**主内容区不能闪烁 / 跳动**

## 5. 信息架构

整个产品就是一个**主页面** + **两个常驻浮层按钮** + **若干模态视图**。

### 5.1 主页面布局（从上到下）

| 区域 | 内容 | 用户目标 |
|---|---|---|
| 词库选择条 | 当前词库 + 难度，可换 | 快速确认 / 切换练习内容 |
| 页面标题 | 应用名 + 题号 | 上下文定位 |
| 播放控制条 | 播放 / 暂停 / 倍速 / 循环 | 控制音频 |
| 句子输入区 | 中文提示 + 逐字格子 + 提示 | 核心练习 |
| 进度区 | 10 个进度标记 | 知道做到第几题 |
| 提交 / 反馈区 | Submit 按钮 / 反馈卡 | 答题 / 看反馈 |

### 5.2 常驻浮层

| 位置 | 元素 | 作用 |
|---|---|---|
| 右上角 | `…` 按钮 | 工具菜单入口 |
| 右下角 | 齿轮按钮 | 快捷键 + 显示选项面板的开关 |
| 底部（自动隐藏） | 上一题 / Esc 提示 / 下一题 | 切题导航 |

### 5.3 模态视图

| 视图 | 触发时机 |
|---|---|
| 加载中 | 拉取任何接口时 |
| 错误页 | 拉取词库 / 句子失败时 |
| 得分页 | 完成所有题，或按 `Esc` 中途查看 |

### 5.4 不应出现的元素

为了保持产品基调，下列元素**不应**出现在主流程中：

- 弹窗广告、推送通知
- 评价请求、问卷调查
- 升级提示、付费引导
- 引导遮罩、新手教学浮层
- 分享按钮、保存按钮（练习是私人的、不分享）

## 6. 交互规则

### 6.1 键盘交互总表

| 键 | 上下文 | 行为 |
|---|---|---|
| `Space` | 任何时候 | 提交 / 跳过空格 / 切换当前格提示 |
| `Enter` | 输入中 | 中间格 → 跳下一格；末位格 + 答对 → 提交 |
| `Tab` | 输入中 | 答对的格跳下一格；未答对的格切换提示 |
| `Shift+Tab` | 输入中（自由模式） | 跳上一格 |
| `Shift+S` | 任何时候 | 切换「显示完整句子」 |
| `Esc` | 弹层未打开 | 跳到得分页 |
| `Esc` | 弹层打开 | 关闭弹层（不跳得分） |
| `←` / `→` | 任何时候 | 上一题 / 下一题 |
| 任意字符 | 输入中 | 清空当前提示 |

### 6.2 输入法兼容

**用户场景**：用中文 / 日文 / 韩文输入法时，会经过「拼音 / 假名 / 谚文」中间态。

**行为规则**（**硬性**）：
- 拼写过程中的临时字符**不污染**用户的输入
- 用户提交（按 Enter 选词）时，**最终字符**进入输入
- 不应出现"明明只是查字典，按完答案错乱"的 bug
- 应兼容主流操作系统 / 浏览器的中文 / 日文 IME

## 7. 容错与匹配规则

**系统对用户答案的宽松匹配**（**硬性**，跨所有模式生效）：

| 规则 | 例子 |
|---|---|
| 大小写不敏感 | `She` ≡ `she` ≡ `SHE` |
| 标点符号忽略 | `quickly,` ≡ `quickly` |
| 连续空白折叠 | `a  big` ≡ `a big` |
| 头尾空白忽略 | ` apple ` ≡ `apple` |

被忽略的标点：`.` `,` `!` `?` `;` `:` `'` `"`

### 9.1 跨设备

## 10. 错误与边界情况

| 场景 | 用户看到 |
|---|---|
| 网络断开 / 后端无响应 | 错误信息（不阻塞其他区域） |

| 用户清除浏览器数据 | 偏好丢失，回到默认值；不影响内容 |

## 12. 文案调性

产品整体的文案风格属于**功能需求**（不是视觉风格）—— 它定义了产品的"声音"。

### 12.1 四条原则

| 调性 | 体现 |
|---|---|
| **禅意** | `dictation.`（句号自带呼吸感） / `a complete breath.` / `quiet progress.` |
| **极简** | 单字 / 双字 + 句号；不感叹、不催促、不评价 |
| **静默** | 答对不弹「🎉 恭喜！」；答错不弹「❌ 再来一次」；状态靠色彩和留白传达 |
| **低姿态** | `begin again` 而不是「再来一局！」；`Press Esc for score` 而不是「查看你的成绩！」 |

### 12.2 正确 / 错误示例

| ✅ 正确 | ❌ 错误 |
|---|---|
| `begin again` | `再来一局！` / `Try Again!` |
| `Press Esc for score` | `查看你的成绩！` |
| `a complete breath.` | `满分！你太棒了！🎉` |
| `7 of 10 — quiet progress.` | `你答对了 70%，继续加油！` |

**所有新文案都应遵循此调性**。如不确定，先用英文写 —— 简短的英文比简短的汉文更易有「呼吸感」。

## 13. 术语表

| 术语 | 含义 |

## 附录 A：当前实现的设计语言（**仅作参考，不是需求**）

> 本附录描述了当前上线版本采用的**视觉与设计语言**。它**不是**本文档的功能需求 —— 如果重新设计或品牌升级，可以**整体替换**而不会违反 §1-§13 中的任何功能约束。
>
> 本节的存在只是为了：**让设计方了解当前实现的状态**，作为"如果什么都不改，长这样"的参考点。

### A.1 美学取向

当前实现采用的是 **Apple HIG（Human Interface Guidelines，web-adapted）** —— Apple 的人机界面规范，强调克制、清晰、单一焦点、单一色彩点。

**五条视觉原则**（在当前实现中体现）：

| 原则 | 当前实现 |
|---|---|
| **Clarity** | 系统字体栈（SF Pro / Segoe UI Variable / Roboto），正文非斜体，文字层级清晰 |
| **Deference** | 单色（中性灰阶），UI 元素不抢戏，主体是句子本身 |
| **Depth** | 用微妙的阴影（`--shadow-sm/md/lg`）和层级（`--surface` / `--surface-secondary`）传达空间感 |
| **Quiet** | 没有旋转、没有笔触动画、没有装饰性纹理；动效是 `ease-standard` 曲线（cubic-bezier(0.25, 0.1, 0.25, 1)） |
| **Single accent** | 朱红 `#D70015` 仅出现在两处：得分页的 enso 円相（品牌印记）+ 正确状态的字符 / cell 文字 |

### A.2 当前的具体视觉选择

> 替换这些**不违反** §1-§13 的任何功能需求。

- **字体**：系统字体栈（`-apple-system, BlinkMacSystemFont, "SF Pro Text", "PingFang SC", "Helvetica Neue", Arial, sans-serif`）+ JetBrains Mono（音标、等宽数字）
- **背景色**：纯白 `#FFFFFF`
- **文字色阶**：Apple 4 级 label 灰阶（`--label-primary` `#1C1C1E` 到 `--label-quaternary` `#C7C7CC`）
- **形态**：8-pt 网格（`--space-1` 4px 到 `--space-12` 128px），圆角 token（`--radius-sm/md/lg/xl/circle`）
- **阴影**：Apple 风格细轻多层（`0 1px 2px rgba(0,0,0,0.04), 0 1px 3px rgba(0,0,0,0.06)` 等）
- **正文**：默认 regular（非斜体）
- **唯一品牌印记**：得分页中央的 120×120 朱红 enso 円相 SVG —— 单笔刷成的圆环，保留 `transform="rotate(-30)"` 的手作姿态

### A.3 当前的具体交互反馈形式

> 这些是**当前选择的具体实现方式**，不是约束。设计师可以换用其他能达到同样效果的形式。

| 反馈类型 | 当前实现 |
|---|---|
| 答对切下一题 | 无延时、立即 |
| 答错 | 显示反馈卡（含正确答案 + Next 按钮），等用户点 Next |
| 切换答案提示 | 淡入 + 4px translateY 上升 |
| 播放按钮 hover | 透明度 0.88 + 阴影加深 + active 时 scale(0.97) |
| 答对音 | E5→G5→C6 三音上行，约 400ms（未改，保留 wabi-sabi 时期的程序化合成的安静旋律） |
| 当前格标识 | `--label-primary` 1.5px 下划线 + 中性灰光标闪烁 |
| 字符对错 | 正确：`--accent` 朱红；错误：`--label-quaternary` + `--label-tertiary` 删除线 |
| 答错提示 | 单元格 0.4s 水平震动 ±3px |
| 进度圆点当前 | `--label-primary` 实心 + 1.4× 放大 |
| 进度圆点答对 | `--accent` 朱红实心 |
| 进度圆点答错 | 透明 + `--label-tertiary` 1px 边框 |
| 快捷键面板齿轮 | 背景 `surface-secondary`，无旋转 |
| Submit 按钮 | 44pt 实心 `--label-primary` + 白字，无下划线无斜体 |
| 答错反馈 Next 按钮 | 1px `--accent` 边框 + 朱红字，hover 实心填充 |
| 得分页 "begin again" | 同上，1px `--accent` 边框 + 朱红字 |
| 答对 wash | 整屏 radial gradient `var(--accent-tint)` 0%→透明 55%，1.2s 渐隐 |
| 答错 wash | 整屏 radial gradient `var(--surface-secondary)` 0%→透明 55%，1.2s 渐隐 |

### A.4 如果要换设计方向，怎么换

可以**独立替换**而不影响功能需求的维度：

| 维度 | 替换为其他风格时的可能性 |
|---|---|
| 美学取向 | Material Design / IBM Carbon / 自定义都可以 —— 只需保留 §9.2 的对比度和 §4.3 的"切换不跳动" |
| 字体 | 任何 —— 但需要保持「正文 vs 音标」的可区分性 |
| 颜色 | 任何 —— 但需要满足 §9.2 的对比度要求 |
| 状态色规则 | 任何 —— 但需要满足 §9.2 的「不依赖单一颜色」要求 |
| 状态色具体颜色 | 必须保留"对 / 错 / 当前 / 激活 / 普通"五种可区分 |
| **enso 品牌印记** | **保留在得分页中央**（§0 已声明）；其他位置可换 |
| 留白密度 | 任何 —— 但要满足 §4.3 的"切换格子时不跳动" |

**不能换**的（属于功能需求）：

- 字符级对错反馈必须存在（§4.3）
- 答对切下一题 + 答错显示反馈卡的双路径必须存在（§4.9.2）
- 答对音必须存在且可关闭（§4.7）
- 输入法兼容必须正常（§6.2）
- 各种持久化偏好必须被记住（§8）
- 得分页 enso 圆环必须保留（§0 品牌印记）
